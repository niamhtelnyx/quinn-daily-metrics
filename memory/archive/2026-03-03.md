## Major Breakthrough: Enhanced Google Drive Integration
**Problem**: Original system only caught calls with strict naming conventions, missed attendee emails in UI chips
**Solution**: Content-based parsing from Gemini summary/details sections

- Email extraction from document titles

## Smart Deduplication Logic
**Key Innovation**: Process Google Drive FIRST, Fellow adds recording URLs later

### Deduplication Flow
1. **Google Drive** (15:35): Full AI analysis + Slack post → Database with dedup_key
2. **Fellow** (16:05): Detects duplicate → Adds Fellow URL to Slack thread
3. **Result**: One analysis, complete recording access, zero duplication

**Dedup Key Format**: `{prospect_email}_{call_date}` (e.g., `roly@meetgail.com_2026-03-03`)

## Enhanced Salesforce Integration 
**Brilliant insight from Niamh**: Search events by meeting subject, not contacts by name

### Event Search Pattern
- **Google Doc**: "Copy of roly@meetgail.com and Ryan: 30-minute Meeting - 2026/03/03 15:59 EST - Notes by Gemini"
- **Core Title**: "roly@meetgail.com and Ryan: 30-minute Meeting"
- **Salesforce**: "Meeting Booked: roly@meetgail.com and Ryan: 30-minute Meeting"

## Salesforce Fallback System
**Unmatched Contacts Table**: Captures all prospects that can't be matched to Salesforce for manual review
- Table: `unmatched_contacts` (prospect_name, email, call_id, source, notes)
- Utility: `check_unmatched_contacts.py` for review/export
